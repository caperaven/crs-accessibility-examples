How to define Window Splitters

A window splitter is a movable separator between two sections, or panes, of a window enabling users to change the relative size of the panes. A window splitter can be either variable or fixed. A fixed splitter toggles between two positions, whereas a variable splitter can be adjusted to any position within an allowed range.

A window splitter has a value that represents the size of one of the panes, which in this pattern, is called the primary pane. When the splitter has its minimum value, the primary pane has its smallest size and the secondary pane has its largest size. The splitter also has an accessible name that matches the name of the primary pane. Note: the term "primary pane" does not describe the importance or purpose of content inside the pane.

Looking at the above example:

  • Two article elements are provided which contain some content, note the div acting as the window-splitter is positioned in between the two article elements.
  • Focusing on the div acting as the separator:
    • It has the separator role indicating a divider separating and distinguishing sections of content (or groups of menu items)
    • The aria-controls attribute is to "main-content" which is the id reference of the first article. This aria property identifies the element whose content or presence is controlled by the current element
    • As no labelling element that would describe the window-splitter is given, the aria-label attribute as been supplied.
    • The aria-valuenow property represents the current position of the separator and is updated as the user interacts with the separator
    • The aria-valuemin property which defines the minimum allowed value.
    • The aria-valuemax property which defines the maximum value for the range.
    • Finally the tabindex is set to 0 to include the separator in the tab page order.

Keyboard Interaction

  • Left Arrow: moves a vertical splitter to the left.
  • Right Arrow: moves a vertical splitter to the right.
  • Up Arrow: moves a horizontal splitter up.
  • Down Arrow: moves a horizontal splitter down.
  • Enter: If the primary pane is not collapsed, collapses the pane. If the pane is collapsed, restores the splitter to its previous position.
  • Home (Optional): Moves splitter to the position that gives the primary pane its smallest allowed size. This may completely collapse the primary pane.
  • End (Optional): Moves splitter to the position that gives the primary pane its largest allowed size. This may completely collapse the secondary pane.
  • F6 (Optional): Cycle through window panes.

NOTE: A fixed size splitter omits implementation of the arrow keys

WAI-ARIA Roles, States & Properties

  • The element that serves as the focusable splitter has role separator
  • The separator element has the aria-valuenow property set to a decimal value representing the current position of the separator.
  • The separator element has the aria-valuemin property set to a decimal value that represents the position where the primary pane has its minimum size. This is typically 0.
  • The separator element has the aria-valuemax property set to a decimal value the represents the position where the primary pane has its maximum size. This is typically 100.
  • If the primary pane has a visible label, it is referenced by aria-labelledby on the separator element. Otherwise, the separator element has a label provided by aria-label
  • The separator element has aria-controls referring to the primary pane.
