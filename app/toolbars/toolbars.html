How to define toolbars

Toolbars act as containers for grouping a set of controls, usually buttons.

The toolbar role can be used to communicate the presence and purpose of the grouped elements to screen reader users. Grouping controls into toolbars is an effective way of reducing the number of tab stops on keyboard. Below is a simple mark-up example of an accessible toolbar implementation.

The following points should be considered to optimize the use of toolbars:

  • Implement focus management. One tab stop for the toolbar, and arrow keys move focus among toolbar controls.
    • In horizontal toolbars, left and right arrow-keys navigate among controls. Up and down arrow-keys can duplicate left and right arrow-keys respectively, or be reserved to operate controls.
    • In vertical toolbars, up and down arrow-keys navigate among controls, while left and right arrow-keys can duplicate the navigation, or be reserved to operate controls.
    • In toolbars with multiple rows of controls, left and right arrow-keys can provide navigation that wraps from row to row, reserving vertical keys for operation.
  • Avoid including controls whose operation requires the pair of arrow keys used for toolbar navigation i.e. don't place a horizontal control within a horizontal toolbar.
  • Use toolbar as a grouping element only if the group contains 3 or more controls.

Keyboard Interaction

  • Tab and Shift+Tab move focus into and out of the toolbar. When focus moves into a toolbar:
    • Focus is set on the first control that is not disabled when moving focus into the toolbar initially.
    • If the toolbar had previously contained focus, focus is either set on the last control that had focus or on the first non-disabled control
  • For horizontal toolbars:
    • Left arrow key moves focus to previous control. Optional: wrap focus from first to last
    • Right arrow key moves focus to next control. Option: wrap focus from last to first
  • Home key(optional), moves focus to first element
  • End key(optional), moves focus to last element
  • For guidance regarding focusability of disabled controls, please refer to WAI-ARIA Authoring Practices 1.1 - 6.7 Focusablity of disabled controls section

WAI-ARIA Roles, States & Properties

  • The element serving as the toolbar has role toolbar.
  • If the toolbar has a visible label, it's reference by aria-labelledby. Else the toolbar element has a label provided by aria-label.
  • If vertical toolbar, toolbar element has aria-orientation set to vertical. Default orientation is horizontal.
