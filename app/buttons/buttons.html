How to define buttons

Buttons enable users to trigger actions/events. In addition to ordinary buttons, WAI-ARIA supports 2 other types:

  • Toggle Button: A two-state button.To tell assistive tech that a button is a toggle button, set aria-pressed attribute. Note: it is important to not change the label when the toggle state changes. If the design calls for the label to change with the state, then the aria-pressed attribute is not needed.
  • Menu Button: Use the aria-haspopup attribute set to either menu or true to tell assistive technologies that it's dealing with a menu button.

Keyboard Interaction

When the button has focus:

  • Space: Activates the button
  • Enter: Activates the button
  • Following button activation, focus is set depending on the type of action the button performs:
    • If activating the button opens a dialog, focus moves into the dialog
    • If activating the button closing the dialog, focus returns to te button that opened the dialog unless the function performed in the dialog leads to a different element
    • If activating the button does not dismiss the current context, the focus usually remains on the button after activation
    • If the button action changes context, then focus is usually moved to the starting point for the action
    • If the button is activated using shortcut keys, focus usually stays within the context from which the shortcut key was activated.
