How to define Multi-thumb Sliders

A multi-thumb sliders is a slider with two or more thumbs that each set a value in a group of related values. Example: in a product search, a two-thumb slider could be used to set the minimum and maximum price limits to filter the search. In many, two-thumb sliders, the thumbs are not allowed to pass one another, such as when the slider sets the min and max values for a range, i.e. the max value of the thumb that sets the lower end of the range is limited by the current value that sets the upper end of the range and vice versa. However in some multi-thumb sliders, each thumb sets a value that does not depend on the other thumb values.

Looking at the example above:

  • The components that make up the slider widget are embedded within the .widget-slider div
  • Note the two .rail-label divs containing the min and max values of the slider at the current moment. These values will be updated accordingly as the user interacts with the slider's left and right thumbs.
  • Looking at the #left-slider-thumb and #right-slider-thumb div elements:
    • Both elements have the slider role which identifies the element as a slider. It is set on the movable thumb because it is the operable element that receives focus and represents the slider value.
    • The tabindex is set to 0 to include the slider thumbs in the page tab sequence
    • The aria-valuemin property is used to indicate the minimum allowed value for each of the thumbs. In this case the right-slider-thumb's minimum value is dependent on the left-slider-thumb's value.
    • The aria-valuemax property is used to indicate the maximum allowed value of each of the thumbs. In this case the left-slider-thumb's maximum value is dependent on the right-slider-thumb's value.
    • The aria-valuenow property is used to indicate the current value of the slider
    • Because the slider is set a filter on price and dealing with currencies, the aria-valuetext is used to indicate the current value of the slider in Rand amounts.
    • Each slider thumb has it's own label set via the aria-label property

Keyboard Interaction

  • Each thumb is in the page tab sequence (has a tabindex of 0) and have the same keyboard interaction as a single-thumb slider
  • The tab order remains constant regardless of thumb value and visual position within the slider i.e. if the value of a thumb changes such that it moves past one of the other thumbs, the tab order does not change.

WAI-ARIA Roles, States & Properties

  • Each element serving as a focusable slider thumb has role slider.
  • Each slider element has the aria-valuenow property set to a decimal value representing the current value of the slider.
  • Each slider element has the aria-valuemin property set to a decimal value representing the minimum allowed value of the slider.
  • Each slider element has the aria-valuemax property set to a decimal value representing the maximum allowed value of the slider.
  • When the range of another slider is dependent on the current value of a slider, the values of the aria-valuemin or aria-valuemax of the dependent sliders are updated when the value changes.
  • If a value of aria-valuenow is not user-friendly i.e. the day of the week is represented by a number, the aria-valuetext property is set to a string that makes the slider value understandable e.g. "Monday"
  • If a slider has a visible label, it is referenced by aria-labelledby on the slider element. Otherwise the slide element has a label provided by aria-label
  • If a slider is vertically oriented, it has aria-orientation set to vertical. The default value of aria-orientation of a slider is horizontal.
