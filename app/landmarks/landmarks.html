What are Landmarks

Using appropriate landmarks help assistive technology understand the overall structure of a web page. Using proper HTML elements like headers, navs, and asides are useful to screen readers. They can collect the landmarks and tell the user what's available on the page. It helps for navigation purposes as the user can jump straight to directly to a footer element, rather that having to navigate to it via tabbing.

Article

The article element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable. A given document can have multiple articles in it, with one or more sections within each article. They are independent in the sense that they could stand alone. However the element is still associated with its ancestors.

They implicitly have the article role. NOTE: an article is not a navigational landmark, but may be nested to form a discussion where assistive technologies could pay attention to article nesting to assist the user in following the discussion.

Please refer to MDN's article definition for a further breakdown of the element.

Aside

The aside element represents content is only indirectly related to the document's main content. They are frequently presented as sidebars or call-out boxes.

They implicitly have the complementary ARIA role applied, which indicates that contained content is relevant to the main content.

Footer

The footer element represents a footer for its nearest sectioning content or sectioning root element.

They implicitly have the contentinfo ARIA role when in the context of the body element. The contentinfo role specifies a large perceivable region that contains information about the parent document. NOTE: The footer element is not a contentinfo landmark when it is a descendant of the following HTML sectioning elements:

  • Article
  • Aside
  • Main
  • Nav
  • section

NOTE: W3 recommends only assigning one element with the contentinfo role within any document or application. However, because document and application element can be nested in the DOM, they may have multiple contentinfo elements as DOM descendants. This is assuming each of those are associated with different document nodes, either by a DOM nesting, or by use of the aria-owns attribute

Form

The form element represents a document section containing interactive controls for submitting information. They have the implicit ARIA role form if the form has an accessible name, otherwise no corresponding role.

The permitted ARIA roles are as follows:

  • Search: which is a landmark region that contains a collection of items and objects that combine to create a search facility.
  • None: which specifies an element whose implicit native role semantics will not be mapped to the accessibility API
  • Presentation: relates to an element which is used to change the look of the page but does not have all the functional interactive, or structural relevance implied by the element, or may be used to provide for an accessible fallback in older browsers which don't support WAI-ARIA

Header

Header elements represent introductory content. It may contain some heading elements but can also include logs, search forms, author names, etc. They implicitly have the role banner, which describes a region that contains mostly site-oriented content, rather than page specific content. When it is a descendant of an article, aside, main, nav, or section element, or an element with role=article, complementary, main, navigation or region then it has no corresponding role.

The permitted ARIA roles are as follows:

  • Group: Which describes a set of user interface objects which are not intended to be included in a page summary or a table of contents by assistive technologies
  • None: which specifies an element whose implicit native role semantics will not be mapped to the accessibility API
  • Presentation: relates to an element which is used to change the look of the page but does not have all the functional interactive, or structural relevance implied by the element, or may be used to provide for an accessible fallback in older browsers which don't support WAI-ARIA

Main

The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application. It has an implicit ARIA role of main which marks the content that is directly related to or expands upon the central topic of the document. It has role attributes permitted.

Nav

The nav element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. It has the implicit role of navigation which describes a collection of navigational elements for navigating the document or related documents. It has role attributes permitted.

Section

The section element represents a standalone section - which doesn't have a more specific semantic element to represent it - contained within an HTML document. They have the implicit ARIA role of region if the element has an accessible name, otherwise no corresponding role

Sections have the following permitted ARIA roles:

  • Alert: A type of live region with important and usually time-sensitive information
  • Alertdialog: A type of dialog that contains an alert message, where initial focus goes to an element with the dialog
  • Application: A structure containing one or more focusable elements requiring user input, such as keyboard or gesture events, that don't follow a standard interaction pattern supported by a widget role
  • Banner: A region that contains mostly site-oriented content, rather than page-specific content
  • Complementary: A support section of the document, designed to be complementary to the main content at a similar level in the DOM hierarchy, but remains meaningful when separated from the main content.
  • Contentinfo: A large perceivable region that contains information about the parent document.
  • Dialog: A dialog is a descendant window of the primary window of a web application.
  • Document: An element containing content that assistive technology users may want to browse in a reading mode.
  • Feed: A scrollable list of articles where scrolling may cause articles to be added to or removed from either end of the list.
  • Log: A type of live region where new information is added in a meaningful order and old information may disappear.
  • Main: The main content of a document
  • Marquee: A type of live region where non-essential information changes frequently
  • Navigation: A collection of navigational elements for navigating the document or related documents.
  • None: which specifies an element whose implicit native role semantics will not be mapped to the accessibility API
  • Note: A section whose content is parenthetic or ancillary to the main content of the resource.
  • Presentation: relates to an element which is used to change the look of the page but does not have all the functional interactive, or structural relevance implied by the element, or may be used to provide for an accessible fallback in older browsers which don't support WAI-ARIA
  • Search: A landmark region that contains a collection of items and objects that combine to create a facility.
  • Status: A type of live region whose content is advisory information for the user but is not important enough to justify an alert.
  • Tabpanel: A container for the resources associated with a tab, where each tab is contained within a tablist
