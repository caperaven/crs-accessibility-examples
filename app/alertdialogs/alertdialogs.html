How to define Alert Dialogs

An alert dialog is a type of dialog that interrupts the user's workflow to communicate an important message acquire a response, e.g. action confirmation prompts, error message confirmations. The alertdialog role enables assistive technologies and browsers to distinguish alert dialogs from other dialogs so they have the option of giving alert dialogs special treatment i.e. playing a system alert sound.

Looking at the example above:

  • The parent container holding the contents and any action elements such as buttons has the attribute role="alertdialog". This indicates a type of dialog that contains an alert message, where the initial focus goes to an element within the dialog.
  • The aria-modal attribute is set to true, indicating the element is displayed as a modal.
  • The alert-labelledby attribute is pointing to the h2 with id="dialog_label" which is used by screen readers to provide a usable name to the user.
  • The aria-describedby attribute is set to the div's id="dialog_desc" which contains a paragraph element describing the alert dialog.
  • NOTE: The last button under the div with id="dialog_actions" has the attribute aria-controls set to "memo_textarea". This attribute identifies the element(or elements) whose contents or presence are controlled by the current element. In this case it is pointing to a memo input which is not displayed in this example, but will have some effect on the input i.e. clearing the contents.

WAI-ARIA Roles, States & Properties

  • The element that contains all elements of the dialog, including the alert message and any dialog buttons, has aria role alertdialog
  • The element with role alertdialog has either:
    • A value for aria-labelledby that refers to the element containing the title of the dialog if the dialog has a visible label
    • A value for aria-label if the dialog does not have a visible label
  • The element with role alertdialog has a value set for aria-describedby that refers to the element containing the alert message
