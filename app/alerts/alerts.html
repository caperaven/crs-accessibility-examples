How to define dialogs

An alert is an element that displays a brief, important message in a way that attracts the user's attention without interrupting the user's task. Dynamically rendered alerts are automatically announced by most screen readers. NOTE: it is important that screen readers do no inform the user of alerts that are present on the page before page load completes.

It is crucial that alerts do not affect keyboard focus in order to maintain the user's ability to continue working. The alert dialog designed for situations where interrupting work flow is necessary.

Avoid designing alerts that disappear automatically. An alert that disappears too quickly can lead to failure to meet WCAG 2.0 success criterion 2.2.3. Another critical design consideration is the frequency of interruption caused by alerts. Frequent interruptions inhibit usability for people with visual and cognitive disabilities, which makes meeting the requirements of WCAG 2.0 success criterion 2.2.4 more difficult

Looking at the example above:

  • The button triggers the corresponding alert.
  • The alert message is wrapped by a parent container with a number of aria related attributes:
    • It has a role of alert, which describes a type of live region with important, usually time-sensitive, information.
    • The aria-live property indicates that an element will be updated, and describes the types of updates the user agents, assistive technologies, and user can expect from the live. When regions are specified as assertive, assistive technologies will immediately notify the user, and could potentially clear the speech queue of previous updates.
    • The aria-atomic property indicates whether assistive technologies will present all, or only parts of, the changed region. If aria-atomic is set to false, assistive technologies will stop searching up the ancestor chain and present only the changed node to the user. IF aria-atomic is set to true, assistive technologies will present the entire contents of the element, including the author-defined live region if one exists.

Keyboard Interaction

  • An alert does not require any keyboard interaction.

WAI-ARIA Roles, States & Properties

  • The widget as a role of alert, which describes a type of live region with important, usually time-sensitive, information.
