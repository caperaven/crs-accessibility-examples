How to define dialogs

A dialog is a descendant window overlaid on either the primary window or another dialog window. You get two types of dialogs:

  • Modal Dialogs: Forces the user to complete/cancel the action before going back to their current workflow. They typically cover a large portion of the screen, and won't let the user interact with content behind the modal dialog.
  • Non-Modal Dialogs: These will still appear on top of the main content, and allow the user to continue to interact with the main content without having to complete/cancel the dialog action.

You can refer to this article on the difference between modal and non-modal dialogs, and when to use them.

Both modal and non-modal dialogs contain their tab sequence, i.e. tab and shit + tab does not move focus outside the dialog. Modal dialogs do not provide means for moving keyboard focus outside the dialog window without closing the dialog.

The alertdialog role is used for dialogs that divert the users' attention to a brief important message. Refer to the alert and message dialog pattern for more on this.

Keyboard Interaction

  • When a dialog opens, focus usually moves to the first element inside the dialog.
    • Focus placement depends on the nature and size of the content.
    • If content is large enough that focusing the first interactive element could cause the beginning of the content to scroll out of view, it is advisable to add tabindex="-1" to a static element at top of the dialog, i.e. dialog title or first paragraph.
    • If a dialog is the last step in a process that is not easily reversible, such as deleting data or completing a financial transaction, it may be advisable to set focus on the least destructive action, especially if undoing the action is difficult or impossible.
    • If a dialog is limited to interactions that either provide additional information or continue processing, it may be advisable to set focus to the element that is likely to be most frequently used, such as an "OK" or "Continue" button.
  • Tab:
    • Moves focus to the next tabbable element inside the dialog.
    • If focus is on the last tabbable element, moves focus to the first tabbable element inside the dialog
  • Shift + Tab:
    • Moves focus to the previous tabbable element inside the dialog
    • If focus is on the first tabbable element, moves focus to the last tabbable element inside the dialog
  • Escape: closes the dialog.

WAI-ARIA Roles, States & Properties

  • The element that serves as the dialog container has role="dialog"
  • All elements required to operate the dialog are descendants of the element that has role dialog
  • The dialog container element has aria-modal="true"
  • The dialog has either:
    • A value set for the aria-labelledby property that refers to a visible dialog title
    • A label specified by aria-label
  • Optionally, the aria-describedby property is set on the element with the dialog role to indicate which element(s) in the dialog contain content that describes the primary purpose or message of the dialog. Specifying descriptive elements enables screen readers to announce the description along with the dialog title and initially focused element when the dialog opens.
