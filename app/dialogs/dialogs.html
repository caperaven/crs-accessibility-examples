How to define dialogs

A dialog is a descendant window overlaid on either the primary window or another dialog window. You get two types of dialogs:

  • Modal Dialogs: Forces the user to complete/cancel the action before going back to their current workflow. They typically cover a large portion of the screen, and won't let the user interact with content behind the modal dialog.
  • Non-Modal Dialogs: These will still appear on top of the main content, and allow the user to continue to interact with the main content without having to complete/cancel the dialog action.

You can refer to this article on the difference between modal and non-modal dialogs, and when to use them.

Both modal and non-modal dialogs contain their tab sequence, i.e. tab and shit + tab does not move focus outside the dialog. Modal dialogs do not provide means for moving keyboard focus outside the dialog window without closing the dialog.

The alertdialog role is used for dialogs that divert the users' attention to a brief important message. Refer to the alert and message dialog pattern for more on this.
